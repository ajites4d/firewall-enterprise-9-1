# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 9.1.x   | ✅                 |
| < 9.1   | ❌                 |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

Email security issues to: **security@firewall-enterprise.com**

You should receive a response within 48 hours.

## Disclosure Process

1. We will confirm the vulnerability
2. Prepare fixes for supported versions  
3. Release patched versions
4. Publicly disclose after patches are available

## Security Best Practices

### For Users
- Always use the latest version
- Keep API keys and secrets secure
- Enable MFA for production deployments
- Regularly rotate encryption keys

### For Developers
- Follow secure coding practices
- Never commit secrets to version control
- Use dependency scanning tools
- Conduct regular security reviews
